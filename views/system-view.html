Give your system a name to be able to recognize it. The description is optional but can be useful to provide more context.

A system is what the provider owns. For example: an API server, a web site, an SDK, etc. A system may have many components but they are irrelevant to the consumers. As far as the consumer is concerned, the provider is responsible to provide a service that they consume to achieve a goal.

At the moment this system does not offer any consumer facing services. Therefore it is not possible to define a service level for it.

The system offers services which are in turn consumed by the consumers.

Examples include: APIs, web sites, app components like SDKs, authetication, etc.
